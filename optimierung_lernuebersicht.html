VL 01
Was ist Optimierung?
Die Suche nach der besten Lösung unter gegebenen Einschränkungen. Ressourcen (Zeit, Geld, Material, Energie) sind begrenzt, Ziele können konkurrieren.
Entscheidungsvariablen
x = (x₁, ..., xₙ)⊤ ∈ ℝⁿ — was wir beeinflussen können (Produktionsmengen, Investitionsanteile, …)
Zielfunktion
f: ℝⁿ → ℝ — was wir minimieren oder maximieren wollen (Gewinn, Kosten, Zeit, …)
Nebenbedingungen
gᵢ(x) ≤ 0, hⱼ(x) = 0 — welche Einschränkungen erfüllt sein müssen (Kapazitäten, Bilanzen, …)
Allgemeine Problemformulierung
min f(x) oder max f(x) u.d.N. gᵢ(x) ≤ 0 (i = 1,...,m) hⱼ(x) = 0 (j = 1,...,p) x ∈ X ⊆ ℝⁿ Zulässiger Bereich: F = {x ∈ X : alle Bedingungen erfüllt} Optimale Lösung: x* mit f(x*) ≤ f(x) ∀ x ∈ F

VL 02
5-Schritte-Modellierungsprozess
1
Problem analysieren — Was soll entschieden werden? Was ist das Ziel? Welche Einschränkungen existieren?
2
Entscheidungsvariablen definieren — Typ, Wertebereich, klare Bedeutung festlegen
3
Zielfunktion formulieren — linear (c⊤x), quadratisch (x⊤Qx), oder nichtlinear (f(x))
4
Nebenbedingungen formulieren — Kapazität, Nachfrage, Bilanz, Nicht-Negativität, Ganzzahl
5
Validieren — Ist das Modell korrekt? Sind alle Einheiten konsistent? Gibt es zulässige Lösungen?

VL 03
Polyeder und geometrische Grundbegriffe
Polyeder
P = {x ∈ ℝⁿ : Ax ≤ b} — Schnitt von endlich vielen Halbräumen. Beschränktes Polyeder = Polytop.
Ecke (Extrempunkt)
Punkt x ∈ P, der nicht als echte Konvexkombination zweier anderer Punkte aus P darstellbar ist. Entspricht einer Basislösung.
Iso-Gewinn-Linie
Menge {x : c⊤x = k} aller Punkte mit gleichem Zielfunktionswert k. Beim Maximieren: in Richtung c verschieben.
Fundamentalsatz der linearen Optimierung
Sei P ⊆ ℝⁿ ein nichtleeres Polyeder und c ∈ ℝⁿ. Für LP max{c⊤x : x ∈ P} gilt:

1. Falls das Maximum endlich existiert, wird es in mindestens einer Ecke von P angenommen.
2. Falls P beschränkt ist (Polytop), existiert immer ein Maximum.

Konsequenz: Es genügt, die (endlich vielen) Ecken zu überprüfen!
Grafisches Lösungsverfahren (2D)
1
Zulässigen Bereich zeichnen — Alle Halbräume einzeichnen, Schnittgebiet markieren
2
Alle Ecken bestimmen — Schnittpunkte der aktiven Restriktionen berechnen
3
Zielfunktion auswerten — c⊤x für jede Ecke berechnen, maximalen Wert wählen
Beispiel: Grafische Lösung
max 3x₁ + 2x₂ u.d.N. x₁ + x₂ ≤ 6 2x₁ + x₂ ≤ 8 x₁ ≤ 3 x₁, x₂ ≥ 0 Ecken und Zielfunktionswerte: (0,0) → z = 0 (3,0) → z = 9 (3,2) → z = 13 (2,4) → z = 14 ← Optimum! (0,6) → z = 12 Optimum: x* = (2, 4), z* = 14
Sonderfälle

VL 04
Motivation
Idee (George Dantzig, 1947): Starte an einer Ecke, wandere zu benachbarten Ecken mit besserem Zielfunktionswert — bis keine Verbesserung mehr möglich. Klappt weil: Zielfunktion linear + Polyeder konvex → jedes lokale Maximum ist global.
Standardform und Schlupfvariablen
LP in Standardform: max c⊤x u.d.N. Ax ≤ b, x ≥ 0 Schlupfvariable sᵢ ≥ 0 wandelt Ungleichung in Gleichung: aᵢ⊤x ≤ bᵢ ⟺ aᵢ⊤x + sᵢ = bᵢ, sᵢ ≥ 0 sᵢ = 0: Restriktion aktiv (Ressource aufgebraucht) sᵢ > 0: Slack / Reserve vorhanden
Simplex-Tableau
Beispiel Möbelfabrik (vereinfacht: x₁=Tische, x₂=Stühle): Basis │ s₁ s₂ x₁ x₂ │ RHS ──────┼──────────────────────┼───── s₁ │ 1 0 6 2 │ 400 s₂ │ 0 1 4 2 │ 300 ──────┼──────────────────────┼───── z │ 0 0 -80 -30 │ 0 z-Zeile negativ → noch nicht optimal x₁ hat Koeffizient -80: viel Potential → Pivotspalte = x₁
Algorithmus — 4 Schritte pro Iteration
1
Optimalitätsprüfung: Alle z-Zeilen-Einträge ≥ 0? → STOPP (optimal)
2
Pivotspalte wählen: Variable mit negativstem z-Koeffizient (Dantzig-Regel) oder kleinstem Index (Bland-Regel, verhindert Cycling)
3
Pivotzeile (Minimum-Ratio-Test): Ratioi = RHSᵢ / aᵢⱼ für alle aᵢⱼ > 0 → kleinste Ratio = Austrittszeile
4
Pivotieren: Pivotspalte durch Zeilenoperationen zur Einheitsspalte transformieren
Schattenpreise
Im optimalen Tableau gibt der z-Zeilen-Koeffizient einer Schlupfvariablen den Schattenpreis an: Wie viel steigt der Zielfunktionswert, wenn Ressource i um 1 Einheit erhöht wird? Schattenpreis = 0 → Restriktion nicht bindend.
Komplexität
Theorie: Worst Case exponentiell (Klee-Minty 1972: 2ⁿ Ecken möglich)
Praxis: Fast immer sehr schnell. Millionen Variablen routinemäßig lösbar.
VL 05
Primales und duales Problem
Primales Problem (P)
max c⊤x u.d.N. Ax ≤ b x ≥ 0
Duales Problem (D)
min b⊤y u.d.N. A⊤y ≥ c y ≥ 0
Dualitätssätze
Schwacher Dualitätssatz: Für jede primal zulässige Lösung x und dual zulässige Lösung y gilt: c⊤x ≤ b⊤y. Jede dual zulässige Lösung liefert eine obere Schranke für (P).

Starker Dualitätssatz: Hat (P) eine optimale Lösung x*, dann hat (D) auch eine optimale Lösung y*, und es gilt: c⊤x* = b⊤y*
Komplementäre Schlupfbedingungen (KKT für LP)
Primale Komplementarität: yᵢ · (bᵢ - aᵢ⊤x*) = 0 ∀ i → Schattenpreis yᵢ = 0 ⟺ Restriktion i nicht bindend Duale Komplementarität: xⱼ · (A⊤y* - c)ⱼ = 0 ∀ j → Variable xⱼ = 0 ⟺ reduzierte Kosten > 0
Ökonomische Interpretation
Die dualen Variablen yᵢ sind die Schattenpreise (Ressourcenpreise): yᵢ gibt an, um wie viel der maximale Gewinn steigt, wenn Ressource i um eine Einheit erhöht wird. Lautet y₁ = 10, würde eine Einheit mehr Holz den Gewinn um 10 € steigern.

VL 06
Mixed-Integer Programming (MIP)
IP — Integer Program
Alle Variablen ganzzahlig: x ∈ ℤⁿ. Notwendig wenn Mengen diskret sind (Anzahl Maschinen, Personen, Einheiten).
MIP — Mixed Integer
Manche Variablen reell, manche ganzzahlig. Sehr häufig in der Praxis (z.B. Standortplanung + Transportmengen).
BIP — Binary IP
xⱼ ∈ {0,1}. Ideal für Ja/Nein-Entscheidungen, Zuweisung, Auswahl aus Alternativen.

LP-Relaxierung und Integrality Gap
LP-Relaxierung: Entferne Ganzzahligkeitsbedingungen → liefert obere Schranke für MIP-Optimum (bei Maximierung) Integrality Gap = (LP-Optimum − IP-Optimum) / LP-Optimum Beispiel Rucksack: LP-Relaxierung: 1900 € (Buch zu 0,75 einpacken) IP-Optimum: 1800 € Integrality Gap ≈ 5,6%
Einfaches Runden der LP-Lösung kann unzulässig sein oder weit vom Optimum entfernt liegen!
MIP ist zwar NP-schwer, aber moderne Solver (Gurobi, CPLEX) lösen viele praktische Instanzen in annehmbarer Zeit — wenn die Modellformulierung gut ist.
VL 07
Branch-and-Bound (B&B)
Grundidee: Teile und herrsche — Zerlege das MIP rekursiv in LP-Teilprobleme, berechne Schranken und schneide hoffnungslose Äste ab (Pruning).
1
Branch: Wähle nicht-ganzzahlige Variable xⱼ = v̄. Erzeuge zwei Teilprobleme: (I) xⱼ ≤ ⌊v̄⌋ und (II) xⱼ ≥ ⌈v̄⌉. Der Streifen ⌊v̄⌋ < xⱼ < ⌈v̄⌉ enthält keine ganzzahligen Punkte.
2
Bound: Löse LP-Relaxierung jedes Teilproblems → obere Schranke (UB) für diesen Ast. Beste bekannte ganzzahlige Lösung = untere Schranke (LB).
3
Prune: Knoten abschneiden wenn: (a) Teilproblem unzulässig, (b) UB ≤ LB (kein besseres Optimum möglich), (c) LP-Lösung schon ganzzahlig → LB aktualisieren
B&B-Algorithmus
Input: MIP, initiale LB = -∞ Initialisiere: Knotenliste L = {Wurzel} While L ≠ ∅: 1. Wähle Knoten k aus L 2. Löse LP-Relaxierung von k 3. Falls unzulässig → verwerfe k 4. Falls LP-Wert ≤ LB → verwerfe k (Bound-Pruning) 5. Falls Lösung ganzzahlig → aktualisiere LB 6. Sonst: Branch auf nicht-ganzzahlige Variable Return: Beste gefundene Lösung
Branching- und Knotenstrategien
Most Fractional
Variable wählen, die am nächsten bei 0.5 liegt. Einfach und oft effektiv.
Strong Branching
Alle Kandidaten testen, besten wählen. Teurer, aber deutlich bessere Bäume.
Depth-First
Findet schnell zulässige Lösungen, geringer Speicherbedarf.
Best-First
Folgt besten Bounds, beweist Optimalität schneller.
Gomory-Schnitte
Aus optimalem Tableau, Zeile i mit nicht-ganzzahliger RHS b̄ᵢ: fⱼ = ā_ij − ⌊ā_ij⌋ (Nachkomma-Anteil Koeffizient) f₀ = b̄ᵢ − ⌊b̄ᵢ⌋ (Nachkomma-Anteil RHS) Gomory-Schnitt: ∑_{j∉B} fⱼ · xⱼ ≥ f₀ → Schneidet LP-Optimum ab, ohne ganzzahlige Punkte zu verlieren!
Branch-and-Cut = B&B + Schnittebenen. Moderne Solver kombinieren beide Ansätze und verwenden zusätzlich problemspezifische Schnitte (Knapsack Cover Cuts, Clique Cuts, Flow Cover Cuts).
VL 08
Klassische kombinatorische Probleme
Rucksackproblem (Knapsack)
max ∑ vⱼxⱼ u.d.N. ∑ wⱼxⱼ ≤ W xⱼ ∈ {0,1}
NP-schwer, aber gut lösbar via B&B, Dynamische Programmierung oder Greedy.
Traveling Salesman (TSP)
Kürzeste Tour durch n Städte, jede genau einmal. (n-1)!/2 mögliche Touren. NP-schwer. Lösbar mit B&B + Subtour-Eliminationsschnitten.
Set Covering
Wähle minimale Anzahl von Mengen, die alle Elemente abdecken. Einsatz: Dienstplanung, Standortwahl, Netzwerkdesign.
Facility Location
Wo eröffnet man Standorte (Fixkosten), um Transportkosten zu Kunden zu minimieren? Klassisches MIP mit Binärvariablen für Standorte.

Approximationsalgorithmen
Wenn exakte Lösung zu teuer: α-Approximationsalgorithmus garantiert eine Lösung, die höchstens α-mal schlechter als das Optimum ist. Z.B. Greedy für Rucksack: 2-Approximation in O(n log n).
VL 09
Nichtlineare Optimierung — Grundlagen
Allgemeines NLP: min f(x) u.d.N. gᵢ(x) ≤ 0 (i = 1,...,m) hⱼ(x) = 0 (j = 1,...,p) x ∈ ℝⁿ Anwendungen: ML (neuronale Netze), Portfolio-Opt., Strukturoptimierung, Regelungstechnik, Chemische Prozessoptimierung
Gradient und Hesse-Matrix
Gradient: ∇f(x) = (∂f/∂x₁, ..., ∂f/∂xₙ)⊤ → zeigt in Richtung steilsten Anstiegs → steht senkrecht auf Niveaulinien Beispiel: f(x₁,x₂) = x₁² + 2x₂² ∇f = (2x₁, 4x₂)⊤ Minimum bei (0,0): ∇f(0,0) = (0,0)⊤ Hesse-Matrix H(x): Hᵢⱼ = ∂²f / (∂xᵢ ∂xⱼ) (Matrix der zweiten partiellen Ableitungen) Beispiel: f(x₁,x₂) = x₁² + 2x₂² H = [[2, 0], [0, 4]]
Optimalitätsbedingungen 1. Ordnung (unbeschränkt)
Notwendige Bedingung: Wenn x* lokales Minimum ist, dann ∇f(x*) = 0 (stationärer Punkt).

Achtung: ∇f = 0 allein reicht nicht! Ein stationärer Punkt kann sein: lokales Minimum, lokales Maximum, oder Sattelpunkt.
Optimalitätsbedingungen 2. Ordnung
Hinreichende Bedingung für lokales Minimum: ∇f(x*) = 0 UND H(x*) ist positiv definit Positiv definit: alle Eigenwerte > 0 ↔ Sylvester: alle führenden Hauptminoren > 0 ↔ Schüsselform: f krümmt sich überall nach oben Positiv semidefinit: alle Eigenwerte ≥ 0 (Grenzfall)
Konvexität
Konvexes Problem: f konvex, gᵢ konvex, hⱼ affin. → Zulässiger Bereich konvex → jedes lokale Minimum ist global. LP ist Spezialfall!
Nichtkonvex: Viele lokale Minima. Startpunkt entscheidet, welches Tal gefunden wird. Neuronale Netze: Verlustfunktion nichtkonvex!
KKT-Bedingungen (mit Nebenbedingungen)
Lagrange-Funktion: L(x, μ, λ) = f(x) + ∑ μᵢgᵢ(x) + ∑ λⱼhⱼ(x) KKT-Bedingungen (notwendig bei geeigneten Regularitätsbedingungen): (1) Stationarität: ∇ₓL(x*,μ*,λ*) = 0 (2) Primal zulässig: gᵢ(x*) ≤ 0 und hⱼ(x*) = 0 (3) Dual zulässig: μᵢ ≥ 0 (4) Kompl. Schlupf: μᵢ · gᵢ(x*) = 0 ∀ i Bei konvexem Problem: KKT-Bedingungen sind notwendig UND hinreichend!
VL 10
Gradientenverfahren
Idee: Nutze den negativen Gradienten als Abstiegsrichtung. Iteriere bis Konvergenz zu einem stationären Punkt.
Gradientenabstieg: x^(k+1) = x^(k) − α_k · ∇f(x^(k)) α_k = Schrittweite (Lernrate) • Zu groß → Divergenz / Oszillation • Zu klein → Sehr langsame Konvergenz • Optimal → Liniensuche erforderlich

Konvergenz
Konvexe Probleme: Konvergenz zum globalen Minimum garantiert. Rate hängt von der Konditionszahl κ = λ_max/λ_min der Hesse-Matrix ab: schlecht konditioniert → zigzag, langsam.
Nichtkonvexe Probleme: Nur Konvergenz zu stationären Punkten garantiert. Kein globales Minimum garantiert. Startpunkt-abhängig.
Praxis: Deep Learning
In Deep Learning ist die Verlustfunktion hochdimensional und nichtkonvex. Trotzdem funktioniert SGD mit Momentum (Adam, RMSprop) sehr gut — da das Netz in der Praxis kein scharfes lokales Minimum benötigt, sondern ein ausreichend flaches.
VL 11
Motivation: Warum ∇f = 0 nicht reicht
Bei beschränkter Optimierung liegt das Optimum oft auf dem Rand des zulässigen Bereichs — dort gilt ∇f(x*) ≠ 0! Der Gradient wird durch die Nebenbedingung „aufgehalten". Wir brauchen neue Werkzeuge.
Lagrange-Multiplikatoren (Gleichungen)
Am Optimum muss ∇f in den Gradienten-Raum der aktiven Nebenbedingungen fallen — d.h. ∇f ist eine Linearkombination der ∇hⱼ.
Problem: min f(x) s.t. hⱼ(x) = 0 (j = 1,...,p) Lagrange-Funktion: L(x, λ) = f(x) − ∑ λⱼ hⱼ(x) Notwendige Bedingung (Stationarität): ∇ₓL = 0 ⟺ ∇f(x*) = ∑ λⱼ ∇hⱼ(x*) hⱼ(x*) = 0 (Zulässigkeit) Interpretation: λⱼ ist der Schattenpreis der j-ten Nebenbedingung. |λⱼ| groß → NB schränkt stark ein; |λⱼ| klein → NB fast irrelevant.
Beispiel: Zwei Gleichungsnebenbedingungen
min x² + y² + z² s.t. x + y = 1, y + z = 1 L = x² + y² + z² − λ₁(x+y−1) − λ₂(y+z−1) Ableitungen = 0: 2x − λ₁ = 0 → x = λ₁/2 2y − λ₁ − λ₂ = 0 → y = (λ₁+λ₂)/2 2z − λ₂ = 0 → z = λ₂/2 Einsetzen in NB: 2λ₁ + λ₂ = 2 und λ₁ + 2λ₂ = 2 → λ₁ = λ₂ = 2/3 → x* = 1/3, y* = 2/3, z* = 1/3
KKT-Bedingungen (Ungleichungen)
Erweiterung der Lagrange-Bedingungen auf Ungleichungsnebenbedingungen gᵢ(x) ≤ 0:
Lagrange-Funktion: L(x, µ, λ) = f(x) + ∑ µᵢ gᵢ(x) + ∑ λⱼ hⱼ(x) KKT-Bedingungen (notwendig): (1) Stationarität: ∇ₓL(x*,µ*,λ*) = 0 (2) Primal zulässig: gᵢ(x*) ≤ 0 und hⱼ(x*) = 0 (3) Dual zulässig: µᵢ ≥ 0 (4) Komplementarität: µᵢ · gᵢ(x*) = 0 ∀ i Komplementarität bedeutet: Entweder die NB ist aktiv (gᵢ = 0) oder ihr Multiplikator ist null (µᵢ = 0) — nie beides ungleich null!
Rezept: KKT-System lösen
1
KKT-Bedingungen aufstellen — Lagrange-Funktion bilden, alle 4 Bedingungen notieren
2
Fallunterscheidung: Für jede Ungleichung: aktiv (gᵢ = 0) oder inaktiv (µᵢ = 0)? → bis zu 2ᵖ Fälle
3
Jedes System lösen und prüfen: µᵢ ≥ 0? Lösung zulässig?
4
Zielfunktionswerte vergleichen aller gültigen Kandidaten
KKT bei konvexen Problemen
Schlüsselregel: Wenn das Problem konvex ist (f konvex, gᵢ konvex, hⱼ affin-linear), dann gilt: KKT-Bedingungen erfüllt ⟺ globales Minimum gefunden. Kein Vergleich mehrerer Kandidaten nötig!
Constraint Qualifications (CQ)
Die KKT-Bedingungen gelten nur unter Regularitätsbedingungen (CQs). Faustregel: Bei linearen NB immer erfüllt. Bei nichtlinearen NB: Gradienten der aktiven NB müssen linear unabhängig sein (LICQ). Ohne CQ können KKT-Bedingungen am Optimum nicht gelten!
Häufige Irrtümer

Motivation: Wann keine Ableitungen?
In vielen Praxisproblemen ist ∇f nicht verfügbar: Simulationen, physische Experimente, Black-Box-Software oder sehr aufwändige numerische Berechnungen. Ableitungsfreie Verfahren optimieren nur durch Funktionsauswertungen f(x).

Nelder-Mead-Simplexverfahren
Arbeitet mit einem Simplex aus n+1 Punkten im ℝⁿ. Der schlechteste Punkt wird in jeder Iteration durch eine der vier Operationen ersetzt:
1. Reflexion
Spiegele schlechtesten Punkt xw am Schwerpunkt x̄ der anderen Punkte: xr = x̄ + (x̄ − xw). Standardfall.
2. Expansion
Gehe noch weiter: xe = x̄ + 2(x̄ − xw). Nur wenn Reflexion sehr gut war (f(xr) < f(xb)).
3. Kontraktion
Halbwegs zu x̄: xc = x̄ + 0.5(xw − x̄). Wenn Reflexion schlechter als zweitschlechtester Punkt.
4. Schrumpfen
Alle Punkte Richtung besten Punkt xb ziehen. Notfall, wenn Kontraktion scheitert.
Nelder-Mead Algorithmus
Initialisierung: Simplex {x₀, x₁, ..., xₙ} (n+1 Punkte in ℝⁿ) Repeat: 1. Sortiere: f(xb) ≤ ... ≤ f(xs) ≤ f(xw) 2. Berechne Schwerpunkt x̄ (alle außer xw) 3. Reflexion: xr = x̄ + (x̄ − xw) 4. Falls f(xb) ≤ f(xr) < f(xs): Ersetze xw durch xr 5. Falls f(xr) < f(xb): Expansion versuchen 6. Falls f(xr) ≥ f(xs): Kontraktion versuchen 7. Falls Kontraktion schlägt fehl: Schrumpfen Until Simplex klein genug (Abbruchkriterium)
Achtung: Nelder-Mead hat keine Konvergenzgarantie — in höheren Dimensionen (n > 10) oder bei rauen Funktionen kann es hängenbleiben oder divergieren.
DIRECT — DIviding RECTangles
DIRECT löst das Problem der globalen Optimierung ohne Ableitungen durch systematische Raumteilung:
1
Teile den Suchraum in Rechtecke und evaluiere f im Zentrum jedes Rechtecks
2
Potenziell optimale Rechtecke identifizieren: Ein Rechteck ist potenziell optimal, wenn kein anderes Rechteck gleichzeitig kleiner UND besseren f-Wert hat
3
Potenziell optimale Rechtecke teilen — entlang der längsten Dimension in drei gleiche Teile
4
Wiederholen bis Budget (maximale Auswertungen) erschöpft ist
DIRECT: Eigenschaften
Vorteile: Keine Ableitungen, keine Startpunkt-Abhängigkeit, keine Parameter zu tunen, garantiert globale Konvergenz (kommt beliebig nah an globales Optimum).
Nachteile: Langsam in hohen Dimensionen (Fluch der Dimensionalität), viele Funktionsauswertungen nötig, Konvergenz kann sehr langsam sein.
Empfohlene Praxis-Strategie
Grob-zu-Fein-Prinzip (kombiniert Global + Lokal): Phase 1 (Global): DIRECT mit begrenztem Budget → Findet vielversprechende Regionen → Identifiziert mehrere lokale Optima-Kandidaten Phase 2 (Lokal): Starte Nelder-Mead von besten DIRECT-Punkten → Schnelle Konvergenz zum nächsten lokalen Optimum → Wenige zusätzliche Auswertungen Ergebnis: Globale Absicherung + lokale Effizienz
